The use of == to compare to objects is expected to do a reference comparison. That is, it is expected to return true if and only if they are the same object instance. Overloading the operator to do anything else will inevitably lead to the introduction bugs by callers. On the other hand, overloading it to do exactly that is pointless; that's what == does by default.

Exceptions

  • If the class overloads the operator + or the operator - then this rule won't raise.
  • If the class implements the IComparable<T> interface then the class most probably behaves as a value-type object so the rule won't raise.
  • If the class implements the IEquatable<T> interface then the class most probably behaves as a value-type object so the rule won't raise.
